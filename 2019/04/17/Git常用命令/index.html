git rebase

  1. 用法1：变基
  1. 用法2：修改（合并）提交信息
  • 执行：git rebase -i 200d4c4（这个commitId是要合并的前一个的提交） ，出现如下交互界面（按i进入插入模式），除了第1行B前面那个pick不要改，C、D前的pick改为s，按esc，再按:wq，保存退出。
  • 再次自动进入交互界面，提示修改提交信息，#开头的为注释，进入插入模式，修改保存退出，提示：Successfully rebased，则合并完成。
  • 查看log，合并完成，最后git push -f推送到远程仓库（如果合并的是已经push的提交）

撤销相关

补充：HEAD最新提交节点，HEADN是HEAD的上N次的版本。这里 [What’s the difference between HEAD^ and HEAD in Git? - Stack Overflow](https://stackoverflow.com/questions/2221658/whats-the-difference-between-head-and-head-in-git) 描述了与HEAD^之间的区别，需要在合并节点上才有讨论意义：

HEAD~N：表示在第1个父节点上位移N步，N等于1时可省略；

HEAD^N：表示在第N个父节点上位移1步，N等于1时可省略；

则：HEAD~~ = HEAD2，HEAD^^ = HEAD2，HEAD^^2表示：在第1个父节点位移1步，再在位移后的第二个父节点位移1步。通常都只是位移1步，用哪个都一样，N值较大时，建议使用commitId，N等于0时，就是HEAD本身。
